Data & Information

Data

Data consists of raw facts and figures without context. It represents raw input that can be processed to generate meaningful insights.

Information

Information is processed data that is meaningful, organized, and useful for decision-making and analysis.

Features of Information

  • Accuracy
  • Relevance
  • Completeness
  • Timeliness
  • Understandability
  • Verifiability

Database and its Purpose

A database is a structured collection of related data stored electronically that allows efficient retrieval, insertion, and management of data.

The purpose of a database includes:

  • Data Storage and Management
  • Data Retrieval
  • Data Security and Integrity
  • Data Sharing and Administration
  • Reducing Data Redundancy

Terminologies in Database

  • Table: A collection of related data organized in rows and columns.
  • Field: A single column in a table representing an attribute.
  • Record: A row in a table representing a single data item.
  • Tuple: Another name for a record; an ordered set of values.
  • Object: An entity with attributes and behaviors in object-oriented databases.
  • Keys: Attributes that help uniquely identify records (Primary Key, Foreign Key).

Data Dictionary

The data dictionary is a centralized repository that contains information about the data such as meaning, relationships to other data, origin, usage, and format.

Database Management System (DBMS)

Introduction

A DBMS is software that manages databases and provides users and applications with a systematic way to create, retrieve, update, and manage data.

Objectives

  • To provide data independence and efficient data access.
  • To support multiple user interfaces.
  • To ensure data integrity and security.
  • To support backup and recovery.

Advantages

  • Improved data sharing
  • Improved data security
  • Better data integration
  • Minimized data inconsistency
  • Improved data access

Disadvantages

  • Costly to set up
  • Complex software
  • May require specialized personnel
  • Performance overhead

Types of Database Models

Hierarchical Model

Data is organized into a tree-like structure where each child has one parent, but parents can have many children.

Network Model

An extension of hierarchical model where each child can have multiple parents, forming a graph structure.

Relational Model

Data is organized in tables (relations) consisting of rows and columns.

Entity-Relational Model (E-R Model)

Uses entities and relationships to represent data and their associations in a more natural manner.

Integrity Constraints and Types

Domain Integrity

Ensures that all entries in a column adhere to a defined domain (data type, format, range).

Entity Integrity

Ensures that each table has a primary key and that it is unique and not null.

Referential Integrity

Ensures foreign keys in a table correctly reference primary keys in related tables.

Keys

  • Primary Key: Uniquely identifies a record.
  • Foreign Key: Enforces a link between tables.

Normalization

Introduction

Normalization is the process of organizing data to reduce redundancy and improve data integrity.

Normal Forms

  1. First Normal Form (1NF): Eliminate repeating groups; atomic column values.
  2. Second Normal Form (2NF): Achieve 1NF and remove partial dependency.
  3. Third Normal Form (3NF): Achieve 2NF and remove transitive dependency.

Advantages

  • Reduces data redundancy
  • Improves consistency
  • Makes database easier to maintain

Disadvantages

  • Can increase complexity with many tables
  • May degrade performance due to many joins

Centralized and Distributed Databases

Introduction

Centralized Database: All data is stored at a single location.

Distributed Database: Data is stored at multiple locations connected via a network.

Advantages

  • Centralized: Easier management and control
  • Distributed: Improved reliability and availability

Disadvantages

  • Centralized: Single point of failure
  • Distributed: Complex to manage and maintain consistency

Comparison

Aspect Centralized DB Distributed DB
Data Location Single site Multiple sites
Management Simpler Complex
Reliability Lower Higher
Cost Less infrastructure More infrastructure

Functions in C Programming

Introduction and Syntax

A function is a self-contained block of code that performs a specific task. Functions help in modularizing code and making it reusable.


// Function prototype (declaration)
return_type function_name(parameter_list);

// Function definition
return_type function_name(parameter_list) {
    // Function body
    // Statements
    return value; // Optional based on return type
}

// Function call
function_name(arguments);

Purpose and Advantages

  • Modularity: Breaks complex problems into smaller, manageable parts
  • Reusability: Functions can be called multiple times
  • Code Organization: Improves readability and maintainability
  • Debugging: Easier to test and debug individual functions

Components of a Function

  • Prototype: Declaration that specifies function name, return type, and parameters
  • Function Definition: Actual implementation of the function
  • Function Call: Invoking the function with actual arguments
  • Return Statement: Returns value from function (void functions don't return)

Types of Functions

  • Library Functions: Predefined functions (e.g., printf(), scanf())
  • User-Defined Functions: Functions created by programmers

Passing Arguments

Call By Value

Original values are not modified - only copies are passed.


#include <stdio.h>

void modifyValue(int x) {
    x = x * 2;
    printf("Inside function: %d\n", x);
}

int main() {
    int num = 5;
    modifyValue(num);
    printf("Outside function: %d\n", num);
    return 0;
}

Call By Reference

Addresses are passed, allowing modification of original values.


#include <stdio.h>

void modifyReference(int *x) {
    *x = *x * 2;
    printf("Inside function: %d\n", *x);
}

int main() {
    int num = 5;
    modifyReference(&num);
    printf("Outside function: %d\n", num);
    return 0;
}

Variable Scope

  • Local Variables: Declared inside a function, accessible only within that function
  • Global Variables: Declared outside all functions, accessible throughout the program

Storage Classes

  • Automatic (auto): Default, local scope, created/destroyed with function calls
  • External (extern): Global variables accessible across files
  • Static: Preserves value between function calls
  • Register: Suggests storing variable in CPU register for faster access

Function with Array Example


#include <stdio.h>

// Function to calculate average of array elements
float calculateAverage(int arr[], int size) {
    int sum = 0;
    for(int i = 0; i < size; i++) {
        sum += arr[i];
    }
    return (float)sum / size;
}

int main() {
    int numbers[] = {10, 20, 30, 40, 50};
    int size = sizeof(numbers) / sizeof(numbers[0]);
    
    float avg = calculateAverage(numbers, size);
    printf("Average: %.2f\n", avg);
    
    return 0;
}

Recursive Functions

Functions that call themselves to solve problems by breaking them into smaller sub-problems.


#include <stdio.h>

// Recursive function to calculate factorial
unsigned long long factorial(int n) {
    if (n == 0 || n == 1) {
        return 1; // Base case
    } else {
        return n * factorial(n - 1); // Recursive call
    }
}

int main() {
    int num = 5;
    printf("Factorial of %d is %llu\n", num, factorial(num));
    return 0;
}

Advantages of Recursion

  • Elegant solution for problems with recursive nature
  • Reduces code complexity for certain algorithms
  • Easier to implement for tree/graph traversals

Disadvantages of Recursion

  • Higher memory usage due to call stack
  • Risk of stack overflow for deep recursion
  • Generally slower than iterative solutions

Structure: Introduction and Syntax

A structure is a user-defined data type that groups related variables of different data types.


// Structure declaration
struct Student {
    int roll_no;

    float marks;

Structure Size

The size of a structure is the sum of sizes of all its members, plus padding for alignment.

Union: Introduction and Syntax

A union is similar to a structure but all members share the same memory location.


#include <stdio.h>

union Data {
    int i;
    float f;
    char str[20];
};

int main() {
    union Data data;
    
    data.i = 10;
    printf("data.i: %d\n", data.i);
    
    data.f = 220.5;
    printf("data.f: %.2f\n", data.f);
    
    // Note: Only one member can contain a value at a time
    return 0;
}

Comparison between Structure and Union

Aspect Structure Union
Memory Allocation Each member has separate memory All members share same memory
Memory Size Sum of all members + padding Size of largest member
Member Access All members can be accessed anytime Only one member active at a time
Use Case Group related data Save memory, type conversion

Pointers in C Programming

Introduction and Syntax

A pointer is a variable that stores the memory address of another variable.


#include <stdio.h>

int main() {
    int var = 10;
    int *ptr; // Pointer declaration
    
    ptr = &var; // Assign address of var to pointer
    
    printf("Value of var: %d\n", var);
    printf("Address of var: %p\n", &var);
    printf("Value of ptr: %p\n", ptr);
    printf("Value pointed by ptr: %d\n", *ptr);
    
    return 0;
}

Usage and Working

  • &: Address of operator
  • *: Dereference operator (value at address)
  • Pointers enable dynamic memory allocation
  • Used for array manipulation and function arguments

Concept of Value and Address

Every variable has both a value and a memory address where it's stored.

Declaration and Initialization


int *ptr;        // Pointer to integer
float *fptr;     // Pointer to float
char *cptr;      // Pointer to character
int **pptr;      // Pointer to pointer to integer

// Initialization
int x = 5;
int *ptr = &x;   // ptr now points to x

Pointer and Function Example


#include <stdio.h>

// Function using pointers
void swap(int *a, int *b) {
    int temp = *a;
    *a = *b;
    *b = temp;
}

int main() {
    int x = 10, y = 20;
    
    printf("Before swap: x = %d, y = %d\n", x, y);
    swap(&x, &y);
    printf("After swap: x = %d, y = %d\n", x, y);
    
    return 0;
}

Call by Reference Example


#include <stdio.h>

void modifyArray(int *arr, int size) {
    for(int i = 0; i < size; i++) {
        arr[i] = arr[i] * 2; // Modify original array
    }
}

void printArray(int *arr, int size) {
    for(int i = 0; i < size; i++) {
        printf("%d ", arr[i]);
    }
    printf("\n");
}

int main() {
    int numbers[] = {1, 2, 3, 4, 5};
    int size = sizeof(numbers) / sizeof(numbers[0]);
    
    printf("Original array: ");
    printArray(numbers, size);
    
    modifyArray(numbers, size);
    
    printf("Modified array: ");
    printArray(numbers, size);
    
    return 0;
}

Comparison: Call by Value vs Call by Reference

Aspect Call by Value Call by Reference
Parameter Passing Copy of value is passed Address/reference is passed
Memory Usage More (creates copies) Less (uses references)
Original Value Not modified Can be modified
Use Case When original shouldn't change When modification is needed

  • Security vulnerabilities if misused

EOF is a constant that indicates the end of a file has been reached.

Typedef Keyword in C

Introduction

The typedef keyword is used to create an alias name for existing data types. It improves code readability and portability.

Syntax


typedef existing_type new_type_name;

Basic Usage Examples


#include <stdio.h>

// Basic type aliasing
typedef unsigned int uint;
typedef float real;

// Structure with typedef
typedef struct {
    int x;
    int y;
} Point;

// Function pointer typedef
typedef int (*MathFunc)(int, int);

int add(int a, int b) { return a + b; }
int multiply(int a, int b) { return a * b; }

int main() {
    // Using basic typedef
    uint count = 10;
    real price = 99.99;
    printf("Count: %u, Price: %.2f\n", count, price);
    
    // Using structure typedef
    Point p1 = {10, 20};
    Point p2 = {30, 40};
    printf("Point 1: (%d, %d)\n", p1.x, p1.y);
    printf("Point 2: (%d, %d)\n", p2.x, p2.y);
    
    // Using function pointer typedef
    MathFunc operation = add;
    printf("Addition: %d\n", operation(5, 3));
    
    operation = multiply;
    printf("Multiplication: %d\n", operation(5, 3));
    
    return 0;
}

Typedef with Arrays


#include <stdio.h>

// Typedef for array types
typedef int IntArray[5];
typedef char String[50];

int main() {
    IntArray numbers = {1, 2, 3, 4, 5};
    String name = "John Doe";
    
    printf("Numbers: ");
    for(int i = 0; i < 5; i++) {
        printf("%d ", numbers[i]);
    }
    printf("\n");
    
    printf("Name: %s\n", name);
    
    return 0;
}

Typedef with Pointers


#include <stdio.h>
#include <stdlib.h>

// Typedef for pointer types
typedef int* IntPtr;
typedef char* StringPtr;

int main() {
    IntPtr numPtr = (int*)malloc(sizeof(int));
    *numPtr = 42;
    
    StringPtr namePtr = "Hello World";
    
    printf("Number: %d\n", *numPtr);
    printf("String: %s\n", namePtr);
    
    free(numPtr);
    return 0;
}

Benefits of Using Typedef

  • Readability: Makes code more understandable
  • Portability: Easier to change underlying types
  • Abstraction: Hides implementation details
  • Consistency: Ensures consistent type usage

Best Practices with Typedef

  • Use descriptive names for type aliases
  • Place typedefs in header files for reuse
  • Use uppercase for type names (convention)
  • Document the purpose of custom types
