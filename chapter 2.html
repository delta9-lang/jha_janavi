Database Questions

1. What is Database and DBMS?

A database is an organized collection of data, generally stored and accessed electronically from a computer system. A Database Management System (DBMS) is software that interacts with end users, applications, and the database itself to capture and analyze data.

Advantages of DBMS:

  • Data Integrity
  • Data Security
  • Data Redundancy Control
  • Improved Data Access

Disadvantages of DBMS:

  • Complexity
  • Cost
  • Performance Overhead
  • Potential for Data Loss

2. Differentiate between file processing system and DBMS.

  • Data Redundancy: File processing systems often have data redundancy, while DBMS minimizes it.
  • Data Integrity: DBMS ensures data integrity, whereas file systems may not.
  • Data Access: DBMS provides better data access methods compared to file systems.
  • Security: DBMS offers more robust security features than file processing systems.

3. Explain the different models of DBMS with advantages and disadvantages.

Common DBMS models include:

  • Hierarchical Model: Advantages: Simple structure, fast access. Disadvantages: Limited flexibility.
  • Network Model: Advantages: More flexible than hierarchical. Disadvantages: Complexity in design.
  • Relational Model: Advantages: Easy to use, powerful querying. Disadvantages: Performance can degrade with large datasets.
  • Object-oriented Model: Advantages: Supports complex data types. Disadvantages: Less mature than relational models.

4. What is a relational database?

A relational database is a type of database that stores data in tables that can be linked—or related—based on data common to each. It differs from other database models by using a structured query language (SQL) for data manipulation.

5. What is data redundancy?

Data redundancy refers to the unnecessary duplication of data within a database. DBMS helps reduce data redundancy by normalizing data and ensuring that each piece of data is stored only once.

6. Differentiate between centralized and distributed database systems.

  • Centralized databases are stored in a single location, while distributed databases are spread across multiple locations.
  • Centralized systems are easier to manage, whereas distributed systems offer better reliability and availability.
  • Centralized databases can become a single point of failure, while distributed systems can continue to function even if one node fails.
  • Distributed databases can provide faster access to data for users in different geographical locations.

7. Who is Database Administrator (DBA)?

A Database Administrator (DBA) is responsible for managing and maintaining database systems. Major responsibilities include:

  • Database design and implementation
  • Performance monitoring and tuning
  • Backup and recovery
  • Security management

8. Define normalization.

Normalization is the process of organizing data in a database to reduce redundancy and improve data integrity. The normal forms include:

  • 1NF: Each column must contain atomic values. Example: A table with student names should not have multiple names in a single cell.
  • 2NF: Must be in 1NF and all non-key attributes must be fully functional dependent on the primary key.
  • 3NF: Must be in 2NF and all attributes must be functionally dependent only on the primary key.

9. Explain the terms: primary key, foreign key, and candidate key.

  • Primary Key: A unique identifier for a record in a table. Example: Student ID.
  • Foreign Key: A field in one table that uniquely identifies a row of another table. Example: Student ID in a grades table referencing the students table.
  • Candidate Key: A field that can uniquely identify a record but is not chosen as the primary key. Example: Email address can be a candidate key.

10. What is SQL?

SQL (Structured Query Language) is a standard programming language for managing and manipulating databases. Its components include:

  • Data Query Language (DQL)
  • Data Definition Language (DDL)
  • Data Manipulation Language (DML)
  • Data Control Language (DCL)

11. Write SQL commands to perform the following operations:

a. Create a table named students:

CREATE TABLE students (Id INT PRIMARY KEY, Name VARCHAR(100), Class VARCHAR(50), Marks INT);

b. Insert records into the students table:

INSERT INTO students (Id, Name, Class, Marks) VALUES (1, 'John', '10', 90);

c. Display all records from the students table:

SELECT * FROM students;

d. Update the marks of a student whose name is 'Anita':

UPDATE students SET Marks = 85 WHERE Name = 'Anita';

e. Delete the record of the student with id = 3:

DELETE FROM students WHERE Id = 3;

12. Define the following terms:

  • Data Dictionary: A collection of metadata that describes the structure of the database.
  • Primary Key: A unique identifier for a record in a table.
  • Relationship: A connection between two tables in a database.
  • Data Manipulation Language (DML): A subset of SQL used for managing data.
  • Structured Query Language (SQL): A standard language for interacting with databases.
  • Data Integrity: The accuracy and consistency of data in a database.
  • Data Definition Language (DDL): A subset of SQL used for defining database structures.
  • Data Security: Measures taken to protect data from unauthorized access.
  • Database System: A system that manages databases and provides access to data.
